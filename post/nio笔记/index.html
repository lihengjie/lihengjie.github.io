Nio笔记

Published 09-19-2019 17:52:13

java NIO 非阻塞IO

可以使用mina框架 NIO三大核心：

Selector

Buffer

Channel

serverSocketChannel.accept()会一直阻塞到有新的连接进入，但是如果提前将ServerSocketChannel.configBlocking(true)这样，accept不会阻塞，会返回null。

selector.selectedKeys()返回一个selectedKeys，（set）

wakeUp() 某个线程调用select()方法后阻塞了，即使没有通道已经就绪，也有办法让其从select()方法返回。只要让其它线程在第一个线程调用select()方法的那个对象上调用Selector.wakeup()方法即可。阻塞在select()方法上的线程会立马返回。

如果有其它线程调用了wakeup()方法，但当前没有线程阻塞在select()方法上，下个调用select()方法的线程会立即“醒来（wake up）”。

close() 用完Selector后调用其close()方法会关闭该Selector，且使注册到该Selector上的所有SelectionKey实例无效。通道本身并不会关闭

Selectort

Buffer

Channel

Channel通道，用来传输数据，类似于火车轨道

Buffer缓冲区，用来从channel中，或者server or client 读写数据，可以想象成火车车厢，（或者是吃饭时，从锅里到碗里，中间的勺子）

Selector很关键，用来监听channel中的事件，比如channel中可以读了，Selector可以监听到OP_READ事件。为了达到让Selector监听的目的，需要将Channel注册到Selector，所以就有了Channel.register(selector,SelectorKey.OP_READ).第二个参数就是我们想让Selector监听的参数， 这个参数可以是一个也可以是多个。这里就需要说一下SelectionKey这个类。 一个的情况：

    public static final int OP_READ = 1 << 0;

    public static final int OP_WRITE = 1 << 2;

    public static final int OP_CONNECT = 1 << 3;

    public static final int OP_ACCEPT = 1 << 4;

如果即想监听connect事件，又想监听accept事件，需要使用或运算符”|“将两个事件连接起来，如下

    Selector selector = Selector.open();
    ServerSocketChannel serverSocketChannel = ServerSocketChannel.open();
    // 自定义监听事件
    int mySelectionKey = SelectionKey.OP_ACCEPT | SelectionKey.OP_CONNECT;
    serverSocketChannel.register(selector, mySelectionKey);

因为我们使用NIO，就是为了不阻塞，所以ServerSocketChannel的实例化对象需要设置为非阻塞模式。

    serverSocketChannel.configureBlocking(false);
    SocketChannel socketChannel = serverSocketChannle.accept();

这个设置之后，在获取连接的时候，serverSocketChannel.accept()的时候，就成了非阻塞模式，如果有连接，直接返回连接，没有就返回null。此外，获取连接可以不在用这种方法， 在迭代keys的时候，如果监听到isAcceptable，就可以在判断中获取到连接。如：

    if (key.isAcceptable()) {// 客户端请求连接
        System.out.println("acceptable开始");
        ServerSocketChannel channel = (ServerSocketChannel) key.channel();
        SocketChannel socketChannel = channel.accept();
    }

当Selector监听到事件后，返回Set，遍历set，判断是读还是写还是accept还是connect，每次遍历，都需要将当前SelectionKey给remove掉，迭代器不会自己移除，下次监听之后会再次将事件加入到该set中。

Buffer缓冲器，其中有capacity，limit，position。flip(),clear(),compact(),mark(),reset(),rewind().equals(),compareTo()

buffer（你看吃饭的勺子，有舀进去，有倒出来）有写模式（舀进去），读模式（倒出来）。

写模式开始时position位于0，limit与capacity一样。当写入一个元素之后，position向后移动一个位置，position一直指向下一次写入的位置，position最大为capacity-1.此时， limit代表可以写的最大长度。

调用flip之后，写模式变为读模式。

此时limit指向position的位置，代表可以读的最大长度。position重置为0。

clear()清空缓存区。

compact()清空已读取的数据，将未读取的数据移至开头，所有新写入的数据追加在未读取的数据之后。

mark()和reset()配合使用，mark标记一个位置，然后读或者写，如果不符合我们预期，调用reset方法，回到刚才mark的位置，重新读写。

rewind()将postion重新指向0，可以重新读取数据。

Selector

Selector.select()返回当前有多少个监听的事件就绪，此方法是阻塞的，如果没有read,write,accept,connect事件就绪，就会一直阻塞，但是，可以使用 Selector.select(3000)或者Selector.selectNow(),一个增加阻塞超时时间，一个立即返回，如果没有就绪事件，返回0.
